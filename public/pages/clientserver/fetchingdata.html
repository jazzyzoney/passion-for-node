Fetching & Redirection: The A-List Communication

Okay, so your front-row look is ready, but how does it interact with the world? It needs to get the latest deets from the server and know exactly where the after-party is. That's where fetching and redirection come in—it's all about that A-list communication.

Fetching Data: Getting the 411

The `fetch()` function is how your webpage texts the server to ask for info. It's the browser's built-in way to make HTTP requests to get data from your API without having to, like, refresh the whole page. So smooth!


// Let's ask the server for the entire outfit collection
fetch("/outfits")
  .then(response => response.json()) // First, get the response and turn it into something usable
  .then(data => {
    // Now you have the data! Time to show it off.
    console.log(data); 
  });

Client vs. Backend Fetching

Usually, the **client** (your browser) fetches from your server. But sometimes, your **backend** needs to fetch from another server (like a weather API). The code looks similar, but the key difference is *who* is making the call. The browser is like you asking for info, while the backend is like your personal assistant making a call on your behalf.

Redirection: Making a Grand Entrance

Redirection is all about moving from one page to another. But are you making the move yourself, or is a VIP escort telling you where to go?

Client-Side: The Quick Outfit Change

This is when the *browser* decides it's time to go to a new page. You use JavaScript to change the URL. It’s perfect for when something happens on the page, like after a user logs in successfully.


// You just logged in? Time for a new look on the dashboard page!
window.location.href = "/dashboard";

Server-Side: The VIP Escort

This is when the *server* tells the browser where to go. The server sends a special response that basically says, "OMG, you can't be here, go over there instead." The browser just follows orders. This is common for page security.


// In your Express server route
res.redirect("/login");
