CRUD Operations: The Head-to-Toe Look

CRUD is the complete package for managing your collection. It stands for Create, Read, Update, and Delete. It’s how you handle every piece from its debut to its final bow.

READ (GET): Checking Out the Collection

A GET request is for when clients want to see your stuff. You can show them the whole collection or just one specific piece if they know its ID.


// Send back all the outfits
app.get("/outfits", (req, res) => {
  res.send({ data: outfits });
});

// Send back the outfit with a specific ID
app.get("/outfits/:id", (req, res) => {
  const outfit = outfits.find(o => o.id === Number(req.params.id));
  res.send({ data: outfit });
});

CREATE (POST): Adding a New Look

A POST request is for adding a brand new look to your collection. The client sends you the details (the `body`), and you add it. To make this work, you *have* to use the `express.json()` middleware. It’s like the translator that lets you understand what the client is sending.


// This middleware is NON-NEGOTIABLE for POST requests!
app.use(express.json());

app.post("/outfits", (req, res) => {
  // req.body has all the deets for the new outfit
  const newOutfit = req.body;
  newOutfit.id = outfits.length + 1; // Give it a new ID
  outfits.push(newOutfit);
  res.send({ data: newOutfit });
});

UPDATE & DELETE: Remixing & Retiring Looks

Once you've mastered GET and POST, updating (PUT) and deleting (DELETE) a look is a total breeze. They follow similar patterns to get a specific item and then either change it or remove it from the collection.
