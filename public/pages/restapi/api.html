Designing Your API: The Style Rules

An API is just a set of rules for communication. A good REST API is like a fashion collection where everything makes sense. There are three major rules to follow:

1. The Actions (HTTP Verbs)

These are the actions you can take with your collection. Always use the right one!

  • GET: Just looking. Used to retrieve data.
  • POST: Adding a new piece to the collection.
  • PUT / PATCH: Changing or updating an existing piece.
  • DELETE: Retiring a look from the collection. Forever.

2. The Labels (Resource Naming)

Your API paths should always be nouns and always be plural. Itâ€™s not `/getOutfit`, itâ€™s `/outfits`. It's not `/accessory`, it's `/accessories`. So much chicer.

3. The Vibe (Ordering & Structure)

The structure should just make sense. To get a specific item, you use its ID. It's like finding a specific dress on a rack.


// Get all the outfits in the collection
GET /outfits

// Get the one specific, iconic outfit with ID 12
GET /outfits/12
