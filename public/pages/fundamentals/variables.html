Variables: Your Style Essentials ✨

Okay, so let's talk about the absolute basics of your code's wardrobe: variables. Think of them like cute little boxes or designer bags where you store all your important stuff—your name, your age, your fave lipstick color. They hold onto your data so you can use it to create a totally stunning look later.

Declaring Variables: The Labels

In fashion, labels matter, and it's the same in code. You've got three main labels for declaring your variables: `const`, `let`, and the totally vintage `var`.

`const`: Your Signature Piece

Use `const` for things that are, like, totally permanent. It's for a value that you're never, ever going to change, like your name or your birthday. Once you set it, it's set for life. So loyal!


// This is my signature style, it's not changing!
const myName = "Yasmin";

// If you try to change it... DRAMA! This will cause an error.
// myName = "Cloe";

`let`: Your Look of the Day

Use `let` when you know your vibe might change. It's for values that can be updated, like your outfit of the day or your current mood. It's flexible and totally gets that sometimes you need to switch things up.


// My mood can totally change.
let myMood = "Fabulous";
myMood = "Extra Fabulous"; // See? Easy update.

console.log(myMood); // Outputs: Extra Fabulous

`var`: So Last Season

OMG, `var` is so vintage. It's from, like, before scoping was even a thing. It can be super confusing and cause major drama in your code. Just stick with `const` and `let`—they're way more in style.

Data Types: The Textures & Fabrics

Variables can hold all different kinds of data, just like your closet has different fabrics. Here's the 411 on the main types.

The Main Clique (Primitive Types)

  • String: Any text, duh. Always wrap it in quotes, like a secret. "Slay!"
  • Number: For any number, like your shoe size or how many lip glosses you own. 25
  • Boolean: The ultimate yes or no. It's either true or false. No in-between.
  • Undefined: When a variable hasn't been given a value yet. It's like an empty shoebox.
  • Null: When you intentionally make a variable have no value. It's like, "I'm not wearing any accessories today."

Type Coercion: The Ultimate Fashion Faux Pas

Type Coercion is when JavaScript tries to guess what you mean and automatically changes a data type. It's like the computer trying to pick your outfit for you—total disaster! It can lead to some seriously weird results.


// JavaScript sees a string and a number and gets confused.
// It decides to just squish them together as a string.
const result = "My age is " + 21; 

console.log(result); // Outputs: "My age is 21" (as a string)

// This is even worse!
const weirdMath = "5" - 2;
console.log(weirdMath); // Outputs: 3 (a number... so confusing!)

Just be, like, super clear with your data types. If you need to combine things, make sure they're the same type first. It's the only way to guarantee a flawless look.
