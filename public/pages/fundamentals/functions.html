Function Declarations

This is the classic, timeless way to define a function. It's like having a little black dress in your closetâ€”it's always in style and you can use it before you even declare it in your code (thanks to hoisting!).

  console.log("Fierce!");
}

Function Expressions

A function expression is a bit more modern and chic. You assign an unnamed (anonymous) function to a variable. It's less "look at me!" until you call it. Unlike declarations, you can't call these before you define them. You gotta get ready before you hit the catwalk.


const serveLook = function() {
  console.log("Serving a flawless look.");
};

// Now you can show it off.
serveLook(); // Outputs: Serving a flawless look.

Parameters & Arguments

Think of **parameters** as the list of accessories a look *needs* (like `outfit` and `shoes`). **Arguments** are the *actual* designer items you use when you create the look (like `"sequin dress"` and `"stilettos"`).


// 'outfit' and 'shoes' are the parameters.
function createLook(outfit, shoes) {
  console.log(`My look is a ${outfit} with ${shoes}.`);
}

// '"vintage top"' and '"platform boots"' are the arguments.
createLook("vintage top", "platform boots");
// Outputs: My look is a vintage top with platform boots.

Callbacks: The BFFs of Functions

A callback is a function you pass into another function as an argument. It's like telling your BFF to call you back *after* she gets the party deets. It's all about timing and making sure things happen in the right order.


function getReady(callback) {
  console.log("Doing my hair and makeup...");
  // After getting ready, the callback function (arriveAtParty) is called.
  callback();
}

function arriveAtParty() {
  console.log("Okay, I've arrived at the party! ðŸŽ‰");
}

// Pass arriveAtParty as the callback.
getReady(arriveAtParty);
